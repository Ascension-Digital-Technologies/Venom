# Support

## Questions and usage help

Use GitHub Discussions when available for architecture, build, compatibility,
and protected-bridge questions. Before asking, review:

- `README.md`
- `examples/protected-chess/README.md`
- `docs/installation.md`
- `docs/configuration.md`
- `docs/PROTECTED-BRIDGE.md`

## Bug reports

Open a GitHub issue using the bug-report form and include:

- Venom version and commit;
- operating system and compiler;
- browser and hosting arrangement;
- exact command line and build profile;
- minimal reproduction;
- console output and runtime verification result.

Do not attach private source code, production secrets, signing keys, or
customer data.

## Security reports

Do not report suspected vulnerabilities publicly. Follow `SECURITY.md` and
send private reports to mario.appledev@gmail.com.

## Supported release line

Support is focused on the latest release line. Generated distributions should
be rebuilt with the current compiler before requesting investigation.

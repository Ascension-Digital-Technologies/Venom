# Browser startup performance

Venom records browser-observable timing evidence as part of the existing Playwright validation path. The measurements are tied to the same exact distribution and behavioral manifest as the compatibility report.

## Recorded milestones

For each browser run, Venom records:

- browser process launch;
- browser-context creation;
- HTTP navigation until Playwright `networkidle`;
- fixture-specific readiness waits;
- declared user-action durations;
- complete scenario duration;
- browser navigation timing values such as DOM content loaded, response end, and load event.

These measurements are intentionally external. They capture what the test runner observes rather than claiming precise internal VM initialization timestamps.

## Evaluate reports

```bash
python tools/browser_performance.py \
  build/browser-validation.json \
  build/static-browser-validation.json \
  --json-out build/browser-performance.json
```

The evaluator reads `contracts/browser-performance.json` and returns exit code `60` when a regression ceiling is exceeded or required timing evidence is absent.

## Interpretation

The checked-in limits are broad CI regression ceilings. They are not end-user latency guarantees and should not be compared across materially different hosts, browsers, power modes, or fixture sets. A passing report does not prove that every application is fast or that complete browser compatibility has been achieved.

For meaningful trend analysis, compare reports generated by the same workflow image, browser versions, fixture manifests, and Venom profile.

# Documentation style guide

- Lead with the user outcome, then explain implementation details.
- Use exact commands that are exercised by tests or release closure.
- Say **raises reverse-engineering cost**, not absolute or permanent-secrecy claims.
- Distinguish browser compatibility, protection, integrity, and secrecy.
- Keep credentials and server-authoritative decisions outside the browser.
- Add audience and version metadata to specialist documents.
- Prefer one canonical page per topic and link to deeper references.
- Update examples, CLI references, and version numbers in the same change as behavior.

# Compatibility evidence

Compatibility claims should be backed by executable fixtures, browser scenarios, console/network assertions, and original-versus-protected behavior comparisons. The repository includes route hydration, worker/runtime, package, bridge, module, malformed HTML, and release-chain regressions. See the legacy evidence index at [COMPATIBILITY-EVIDENCE.md](../COMPATIBILITY-EVIDENCE.md).


The canonical original-versus-protected workflow is documented in [browser equivalence testing](browser-equivalence.md).

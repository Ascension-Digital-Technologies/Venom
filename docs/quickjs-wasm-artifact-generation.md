# QuickJS WASM artifact generation and verification — v0.88.0

Venom v0.88.0 makes the QuickJS WASM cutover path verifiable instead of relying on release metadata claims. The Emscripten build now produces a `.wasm`, inspects its actual WebAssembly export section, checks it against the ABI12 C export declarations, writes a manifest, and embeds that manifest into the generated C++ blob header as provenance.

## v0.88.0 preflight layer

Before compiling, the build scripts now run `tools/quickjs_wasm_preflight.py`. It writes `quickjs-wasm-preflight.txt` and `quickjs-wasm-preflight.json`, verifies the vendored QuickJS files and ABI12 export declarations, and reports whether Emscripten is available.

Use the safe repo-only check when Emscripten is not installed yet:

```bash
scripts/build-quickjs-wasm.sh --preflight-only
```

PowerShell:

```powershell
scripts\build-quickjs-wasm.ps1 -PreflightOnly
```

## What this pass adds

- `tools/wasm_exports.py` for direct WebAssembly export-section inspection
- upgraded `tools/embed_wasm.py` with provenance embedding
- `quickjs-runtime.exports.json` with actual exports and required ABI exports
- `quickjs-runtime.manifest.txt` with the real-engine truth markers
- package metadata propagation from the embedded provenance record
- stricter `verify-runtime --require-real-engine` checks:
  - `artifact_kind=upstream-quickjs-wasm`
  - `required_exports_satisfied=true`
  - `missing_export_count=0`
  - no scaffold/runtime-contract-only markers


## Current truth state

The default checked-in blob remains the contract scaffold unless `scripts/build-quickjs-wasm.*` is run and the generated header is committed. Therefore:

```text
venom verify-runtime dist --target browser
```

passes for the current contract-scaffold release metadata, while:

```text
venom verify-runtime dist --target browser --require-real-engine
```

fails until the generated upstream QuickJS WASM blob is embedded.

Expected scaffold markers:

```text
quickjs_runtime_implementation: quickjs-wasm-contract-scaffold
quickjs_runtime_artifact_kind: contract-scaffold
quickjs_runtime_required_exports_satisfied: no
quickjs_runtime_finish_blocker: run_build_quickjs_wasm_and_embed_verified_artifact
```

Expected verified generated-WASM markers:

```text
quickjs_runtime_implementation: quickjs-wasm-upstream-quickjs
quickjs_runtime_artifact_kind: upstream-quickjs-wasm
quickjs_runtime_required_exports_satisfied: yes
quickjs_runtime_missing_export_count: 0
quickjs_runtime_real_engine_candidate: yes
quickjs_runtime_full_upstream_quickjs: yes
```

## Cutover rule

A release may only pass `verify-runtime --require-real-engine` when all of these are true:

1. `src/generated/runtime/quickjs_runtime_wasm_blob.hpp` was generated by `scripts/build-quickjs-wasm.*`.
2. `tools/wasm_exports.py` confirms every ABI12 export in `src/runtime/quickjs_runtime_scaffold.c` exists in the generated `.wasm`.
3. The embedded provenance says `required_exports_satisfied=true` and `missing_export_count=0`.
4. Browser `verify-runtime --require-real-engine` passes.
5. DOM, console, timer, fetch, event, module, exception, promise, and route fixtures pass without host source-eval fallback.
